# FinOS App (Flutter)

Mobile-first, offline-first client. See [../ARCHITECTURE.md](../ARCHITECTURE.md) §5.

## Structure (feature-first)

```
lib/
├── main.dart                 # app entry (ProviderScope + MaterialApp.router)
├── core/
│   ├── theme/                # AppColors, AppTheme (Material 3)
│   └── router/               # GoRouter (Routes + routerProvider)
├── features/
│   ├── onboarding/presentation/   # SplashPage, WelcomePage (the opening page)
│   └── auth/presentation/         # SignInPage (Supabase Auth lands in Phase 0)
└── shared/
    └── widgets/              # FinosLogo and other reusable UI
```

As features arrive (Phase 1+) each gets `data/ domain/ application/ presentation/`
under `features/<name>/`.

## First-time setup

Platform folders (`android/`, `ios/`, …) are generated by Flutter, not committed here.
After installing Flutter:

```bash
cd frontend
flutter create . --project-name finos --org com.finos   # generate platform scaffolding
flutter pub get
flutter run                                              # launch the opening page
```

## Quality gates

```bash
flutter analyze
flutter test
```

## Design notes

- **Opening page** (`welcome_page.dart`) leads with the promise — *understand your money* —
  and three pillars (Track → Plan → Decide) that mirror the product spine. It carries the
  privacy + "informational, not advice" line by design (see [../SECURITY.md](../SECURITY.md)).
- Money will always render through a shared `Money` type + INR formatter — never raw doubles.
- UI reads from repositories backed by the local DB (Drift); widgets never call the network.
